# User Story #1 — 클라이언트와 REST API 연결 및 "Hello World" 응답 (Server)

> 출처: 프로젝트 슬라이드 "클라이언트와 서버간 Rest API를 이용한 연결 만들어 통신 하기"
> 범위: **이 환경은 Server 전용**(CLAUDE.md §0). 아래는 슬라이드에서 **서버가 수행해야 하는 책임**만 추출한 것이며, 클라이언트 동작(연결 요청 발신·문자열 출력 등)은 인터페이스 관점(수신 계약)으로만 참조한다.
> 인터페이스: Control 채널 = **REST** (CLAUDE.md §3.1).

---

## Story

**As a** Server,
**I want to** 클라이언트의 REST 요청으로 세션을 맺고, 전달받은 문자열 `"Hello"`에 `"World"`를 붙여 `"Hello World"`로 응답하며, 요청 시 세션을 해제할 수 있도록
**so that** 클라이언트와 서버 간 REST API 통신 및 세션 생명주기(접속 → 통신 → 해제)를 검증할 수 있다.

---

## 서버 책임 (슬라이드에서 추출)

| # | 서버가 할 일 | 슬라이드 원문 근거 |
|---|--------------|--------------------|
| 1 | 클라이언트의 연결 요청을 받아 **해당 클라이언트와 연결(세션)을 생성**한다 | "서버가 해당 클라이언트와 연결을 만들고" |
| 2 | 클라이언트가 보낸 문자열 `"Hello"`를 **수신**한다 | "클라이언트에서 스트링 ''Hello''를 서버에 전송" (수신 측) |
| 3 | 수신한 `"Hello"`에 `"World"`를 추가해 `"Hello World"`를 **클라이언트로 전송(응답)**한다 | "서버에서 수신된 스트링 ''Hello''에 ''World''를 추가해 클라이언트에 전송" |
| 4 | 클라이언트의 **접속 해제 요청을 받아 해당 클라이언트 세션을 해제**한다 | "서버가 해당 클라이언트 접속 해제" |

### 범위 밖 (클라이언트 담당 — 이 환경에서 구현하지 않음)
- 클라이언트가 서버에 연결을 요청하는 행위(발신)
- 클라이언트가 수신한 `"Hello World"`를 화면에 출력하는 행위
- 클라이언트가 접속 해제를 요청하는 행위(발신)

> 위 항목들은 서버 입장에서는 "수신 이벤트"로만 다룬다.

## 인수 조건 (Acceptance Criteria)

- [ ] **AC1 (연결)** 클라이언트의 연결 요청 REST 호출 시, 서버가 세션을 생성하고 세션 식별자(예: `sessionId`)와 성공 응답을 반환한다.
- [ ] **AC2 (에코 규칙)** 요청 본문/파라미터가 정확히 `"Hello"`일 때, 서버는 `"Hello World"`(= 수신 문자열 + `" World"`)를 응답한다.
- [ ] **AC3 (세션 확인)** 유효하지 않거나 이미 해제된 세션으로 요청하면 서버가 오류(예: 404/400 등 적절한 상태 코드)를 반환한다.
- [ ] **AC4 (해제)** 클라이언트의 접속 해제 요청 시, 서버가 해당 세션을 제거하고 성공 응답을 반환한다. 이후 같은 세션 사용은 AC3에 따라 거부된다.
- [ ] **AC5 (테스트)** 위 시나리오(연결 → "Hello" 전송 → "Hello World" 수신 → 해제)를 검증하는 JUnit 테스트가 존재하고 통과한다. (CLAUDE.md §5.3)

## 제안 인터페이스 (초안 — 구현 시 확정)

> Control 채널은 REST(CLAUDE.md §3.1). 아래 경로/스키마는 초안이며 구현 단계에서 확정한다.

| 동작 | Method | Path | 요청 | 응답(예) |
|------|--------|------|------|----------|
| 연결 | `POST` | `/api/session` | — | `201 { "sessionId": "<id>" }` |
| 통신(에코) | `POST` | `/api/session/{sessionId}/echo` | `{ "message": "Hello" }` | `200 { "message": "Hello World" }` |
| 해제 | `DELETE` | `/api/session/{sessionId}` | — | `204` |

## 미결정/확인 필요
- 다중 클라이언트 세션 동시 지원 여부 (CLAUDE.md §6-1 미결정 이슈와 연결) — 세션 저장소 설계에 영향.
- 에코 규칙의 일반화 범위: `"Hello"`만 특수 처리할지, 임의 문자열에 `" World"`를 붙일지. (슬라이드는 `"Hello"` 기준)
